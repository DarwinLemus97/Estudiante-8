Conociendo Open GL

Es un lenguaje cruz , cruz-plataforma de interfaz de programación de aplicaciones (API) para la representación en 2D y 3D de gráficos vectoriales . El API se utiliza típicamente para interactuar con una unidad de procesamiento gráfico (GPU), para lograr acelerado por hardware de renderizado. Silicon Graphics Inc. (SGI) comenzó a desarrollar OpenGL en 1991 y lo liberó el 30 de junio de 1992; [5] [6] aplicaciones utilizan ampliamente en los campos de diseño asistido por ordenador (CAD), la realidad virtual , visualización científica , visualización de información, simulación de vuelo y juegos de video . Desde 2006 OpenGL ha sido gestionado por la sin fines de lucro la tecnología consorcio Grupo Khronos.

Desarrollo

OpenGL es una API en evolución. Las nuevas versiones de las especificaciones de OpenGL se liberan regularmente por el Grupo Khronos , cada uno de los cuales se extiende la API para soportar varias nuevas características. Los detalles de cada versión se deciden por consenso entre los miembros del Grupo, incluidos los fabricantes de tarjetas de gráficos, diseñadores de sistemas operativos, y las empresas de tecnología generales, tales como Mozilla y Google.

Historia

En la década de 1980, el desarrollo de software que podría funcionar con una amplia gama de hardware gráfico era un verdadero desafío. Los desarrolladores de software escribieron interfaces y controladores personalizados para cada pieza de hardware. Esto era caro y dio lugar a la multiplicación de esfuerzos. A comienzos de 1990, Silicon Graphics (SGI) fue un líder en gráficos 3D para estaciones de trabajo. Su IRIS GL API [13] se considera el estado de la técnica [ cita requerida ] y se convirtió en el estándar de facto del sector, eclipsando los basados en estándares abiertos PHIGS . Esto se debió a IRIS GL se considera más fácil de usar, y porque apoyaba el modo inmediato prestación. Por el contrario, PHIGS se consideró difícil de usar y anticuado en la funcionalidad.

Diseño

La especificación OpenGL describe un resumen de la API para la elaboración de gráficos 2D y 3D. Aunque es posible que el API para ser implementado en su totalidad en el software, que está diseñado para ser implementado principalmente o exclusivamente en el hardware . El API se define como un conjunto de funciones que pueden ser llamados por el programa cliente, junto con un conjunto de constantes enteras con nombre (por ejemplo, el GL_TEXTURE_2D constante, que corresponde a la decimal número 3553). Aunque las definiciones de funciones son superficialmente similares a las del lenguaje de programación C , que son independiente del lenguaje. Como tal, OpenGL tiene muchos enlaces de lenguaje , algunos de los más notables es el JavaScript vinculante WebGL (API, basado en OpenGL ES 2.0 , para la representación 3D desde dentro de un navegador web ); los enlaces C WGL , GLX yCGL ; el C unión proporcionada por iOS ; y los de Java fijaciones y C proporcionados por Android.
